# Gemini颜色（基于物理）

**评分: ⭐⭐⭐**

## 概念

使用Gemini Vision API咨询来确定基于温度分布的物理准确火焰颜色。避免HSV→BGR转换伪影（蓝边问题）。

## Gemini提供的BGR值

基于火焰温度物理：

```python
# BGR格式（OpenCV）
colors = {
    'core':       (255, 255, 255),  # 白色 - 过曝
    'inner':      (0, 255, 255),    # 黄色 - 最热可见区
    'mid':        (0, 165, 255),    # 橙色 - 中温
    'outer':      (0, 69, 255),     # 深橙色 - 较冷
    'tips':       (0, 0, 200),      # 红色 - 最冷
}
```

## 为什么重要

早期迭代使用HSV颜色空间，转换回BGR时有**蓝边伪影**。直接指定BGR消除了这个问题。

## 温度区域

| 区域 | 温度 | 外观 |
|------|------|------|
| 核心 | >1400°C | 白色（过曝） |
| 内层 | 1100-1400°C | 亮黄色 |
| 中层 | 800-1100°C | 橙色 |
| 外层 | 500-800°C | 红橙色 |
| 尖端 | <500°C | 暗红色 |

## 整合

这些颜色值现已整合到其他方法（云状、垂直等），以获得一致的、物理合理的外观。
